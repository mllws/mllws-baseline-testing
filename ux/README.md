# ux/

Lighthouse Best Practices + SEO runs, Playwright responsive screenshot script (375/768/1440px), Playwright scripted conversion-flow walkthrough. See issue: "Set up UX baseline tooling."

- `ux-config.json` — locked 7 URLs, screenshot widths (`375`/`768`/`1440`), Chrome launch config.
- `run-screenshots.js` — Playwright full-page screenshot at each width for every URL, saved to `screenshots/<slug>/<width>px.png`; writes the index to `../reports/ux-screenshots.md`. Run with `npm run ux:screenshots`.
- `run-flow.js` — Playwright walkthrough of the conversion-flow proxy locked in `docs/PLAN.md` Phase 1 (Contact + Volunteer — the site has no functional form anywhere): loads Contact, checks `mailto:`/`tel:` links, loads Volunteer, checks the "Join us!" button is visible, clicks it and asserts it does not navigate (confirms the known dead-link gap rather than papering over it). Records step timing + pass/fail to `results/flow-run.json` and `../reports/ux-flow.md`. Run with `npm run ux:flow`.
- `run-lighthouse-bp-seo.js` — Lighthouse Best Practices + SEO categories per URL, saves JSON to `results/`, writes `../reports/ux-baseline.md`. Run with `npm run ux:lighthouse`.

Screenshot archive lives in `ux/screenshots/`; other raw results live in `ux/results/`.

Tooling only — no baseline has been run yet. `screenshots/`, `results/`, and the `../reports/ux-*.md` files are generated by the three `npm run ux:*` scripts when the actual baseline pass happens.
